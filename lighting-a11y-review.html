Accessibility review

Start with the keyboard

Aim to have a logical tab order and easily discernable focus styles

Key points

Start by tabbing through your site

Interactive (native and custom) controls should aim to be focusable and display a focus indicator

Avoid controls with a tabindex > 0

Non-interactive content (e.g., headings) should avoid being focusable

Direct focus to new content added to a page

Beware of completely trapping focus at any point

Don’t forget offscreen content

Try it with a screen reader

Check the page for proper labeling and semantics and any obstructions to screen reader navigation

Key points

Check all images for proper alt text

Check all controls for a label. Look at aria attributes for custom controls

The flow of information should make sense

Aim to support a screen reader’s navigation to all content on the page

For content that needs to be hidden, make sure that is set to aria-hidden=”true”

Headings and landmark elements give your page semantic structure, and greatly increase the navigating efficiency of assistive technology users
